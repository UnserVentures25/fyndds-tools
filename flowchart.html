1
Original-Kauf & Lieferung
Normaler E-Commerce — FYNDDS ist noch nicht involviert
01 Käufer A
Käufer A kauft Produkt beim Händler
Ganz normaler Online-Kauf. Käufer A bestellt auf dem Händler-Shop (Shopify, WooCommerce etc.) und bezahlt. FYNDDS ist hier noch gar nicht beteiligt.
02 Händler + DHL
Händler versendet das Paket via DHL
Händler erstellt ein DHL-Label und übergibt das Paket. DHL liefert an Käufer A.
03 Käufer A
Käufer A möchte zurückgeben
Käufer A ist unzufrieden — falsches Produkt, passt nicht, Geschmack geändert. Er klickt auf "Retoure anmelden" im Händler-Shop.
💡 Genau hier setzt FYNDDS an. Anstatt das Paket einfach einzulagern, wird es über FYNDDS weiterverkauft.
2
Return-Portal — Käufer A gibt Paket auf
FYNDDS übernimmt den Rücksende-Prozess
04 FYNDDS
Händler schickt Käufer A auf das FYNDDS Return-Portal
Der Händler hat das FYNDDS Return-Portal eingebunden (als White-Label-Seite mit eigenem Logo). Käufer A bekommt einen Link, der ihn auf eine Seite führt unter z.B.: returns.haendler.de
Was Käufer A auf dem Portal ausfüllt:
→ Warenzustand (wie neu / gut / akzeptabel)
→ Retourengrund (passt nicht, Beschreibung falsch, etc.)
→ Wie lange er die Ware noch hat (Sales-Window-Bestätigung)
→ Optional: Fotos hochladen
05 FYNDDS Engine API
FYNDDS holt Produktdaten vom Händler-Shop
FYNDDS ruft die Händler-API auf (oder 8returns-Webhook) und holt: Produktname, Kategorie, Original-Preis, Bilder, SKU. Damit wird das Listing für den FYNDDS-Marketplace vorbereitet.
06 FYNDDS + DHL DHL Shipping API
FYNDDS erstellt DHL-Retourenlabel unter eigenem EKP
Das ist der kritische Schritt für das spätere Rerouting. FYNDDS erstellt das Label über die eigene DHL-Geschäftskunden-Nummer (EKP). Käufer A bekommt einen QR-Code.
⚠️ Warum FYNDDS-EKP? Nur weil das Label unter FYNDDS läuft, kann FYNDDS später die Zieladresse des Pakets ändern. Ein Label das der Händler erstellt hat, kann FYNDDS nicht umeroutern.
07 Käufer A + DHL
Käufer A gibt Paket bei DHL ab
Käufer A druckt das Label oder zeigt den QR-Code an der DHL-Filiale. Das Paket ist jetzt offiziell unterwegs. Die Zieladresse im Label ist die FYNDDS-Retourenadresse — vorerst.
3
Paket unterwegs → FYNDDS listet das Produkt
Das Sales-Window öffnet sich — die Uhr tickt
08 DHL → FYNDDS Webhook
DHL meldet: "Paket abgeholt"
DHL scannt das Paket bei der Filiale ein. Automatisch schickt DHL einen Webhook (HTTP-POST) an FYNDDS: "Sendung XY wurde abgeholt."
FYNDDS tracking-webhook empfängt das Event:
Signatur wird geprüft (HMAC-SHA256) → Event wird verarbeitet
09 FYNDDS Engine
Sales-Window öffnet sich — Produkt erscheint auf Marketplace
FYNDDS setzt den Asset-Status auf listed und startet den 48-Stunden-Timer. Das Produkt erscheint sofort im Marketplace mit einem dynamischen Preis.
Dynamisches Pricing:
→ Startpreis: 12% unter Original-Preis
→ Preis fällt automatisch alle ~12h weiter
→ Nach 48h: max. 35-40% Rabatt
→ Je früher Käufer B kauft, desto höher der Preis für den Händler
4
Käufer B kauft auf FYNDDS
Stripe-Zahlung + Order-Erstellung
10 Käufer B
Käufer B findet das Produkt und kauft
Käufer B stöbert im FYNDDS Marketplace, sieht das Produkt mit Countdown-Timer und Rabatt, legt es in den Warenkorb und geht zur Kasse.
11 Stripe Stripe Checkout
Käufer B bezahlt via Stripe
Käufer B gibt Lieferadresse ein und zahlt. Stripe verarbeitet die Zahlung.
Was Käufer B zahlt:
→ Produktpreis (z.B. €72,00)
→ + Rerouting-Gebühr €2.99 (bleibt bei FYNDDS)
→ = Total: €74.99

Wie das Geld aufgeteilt wird (Stripe Connect):
→ FYNDDS-Provision: Min(€15, Max(€2.50, Produktpreis × 3%)) = z.B. €2.16
→ Händler bekommt: €72.00 − €2.16 = €69.84 (nach Zustellung)
12 FYNDDS Engine stripe-webhook
Stripe bestätigt Zahlung → FYNDDS erstellt Order
Stripe schickt einen Webhook an FYNDDS: "Zahlung erfolgreich." FYNDDS-Engine erstellt in der Datenbank eine Order mit Bestellnummer, Käufer B's Lieferadresse, und setzt das Asset auf "sold".
5
Das Rerouting — der Kern von FYNDDS
DHL ändert die Zieladresse des Pakets mid-transit
13 FYNDDS Engine initiate-rerouting
FYNDDS prüft ob Rerouting noch möglich ist
Bevor FYNDDS DHL anruft, fragt es zuerst: Wo ist das Paket gerade?
Status-Check:
✅ in_transit / pre-transit → Rerouting möglich, weiter
❌ out_for_delivery / delivered → Zu spät! → Fallback aktivieren
14 FYNDDS → DHL DHL PATCH /address
FYNDDS ändert die Zieladresse beim Paket
FYNDDS schickt einen PATCH-Request an DHL mit der neuen Adresse von Käufer B. DHL ändert intern die Routing-Daten.
DHL API-Call:
PATCH /parcel/de/v1/shipments/{trackingNr}/address
Body: name, Straße, PLZ, Stadt von Käufer B

DHL antwortet: 200 OK → Rerouting bestätigt ✅
14b Fallback (wenn Rerouting nicht klappt)
Paket kann nicht mehr umgeleitet werden
Passiert selten, aber möglich: Paket ist schon beim Fahrer oder zugestellt. DHL antwortet mit 409 (Conflict).
Was FYNDDS automatisch tut:
1. Bestellung von Käufer B wird storniert
2. Stripe Refund wird ausgelöst (Geld zurück)
3. Asset wird wieder gelistet (+24h Verlängerung)
4. Käufer B bekommt E-Mail: "Leider nicht möglich, Rückerstattung läuft"
6
Lieferung & Auszahlung
Käufer B bekommt sein Paket — Händler bekommt sein Geld
15 DHL
DHL liefert das Paket an Käufer B
Das Paket, das ursprünglich auf dem Weg zurück zum Händler war, landet jetzt direkt bei Käufer B. Kein Lager, kein Umweg, kein Aufwand für den Händler.
16 DHL → FYNDDS Webhook "delivered"
DHL meldet: "Zugestellt" → Auszahlung wird getriggert
DHL schickt automatisch einen Webhook an FYNDDS: "Paket wurde zugestellt." FYNDDS-Engine startet sofort die Auszahlung an den Händler.
17 Stripe Connect process-payout
Auszahlung an den Händler
Stripe überweist den Händler-Anteil direkt auf das Stripe-Konto des Händlers.
Abrechnung (Beispiel: Produkt €72):
→ Verkaufspreis: €72.00
→ FYNDDS-Provision (3%): − €2.16
→ Händler bekommt: €69.84 auf sein Stripe-Konto
→ FYNDDS behält: €2.16 (Provision) + €2.99 (Rerouting-Fee) = €5.15
18 Händler
✅ Fertig — Händler hat Geld erhalten
Der Händler hat seine Retoure erfolgreich monetarisiert — ohne Lagerkosten, ohne Aufwand, ohne manuellen Eingriff. Das Paket ist direkt von Käufer A zu Käufer B geflossen.
Was der Händler gespart hat:
→ Keine Lagerkosten für die Retoure (~€8-15/Einheit)
→ Kein Aufwand für Qualitätsprüfung & Wiederverkauf
→ Kein CO₂ durch Lagerumweg
→ Geld nach Zustellung automatisch auf dem Konto
Zusammenfassung
Was FYNDDS in einem Satz macht
Käufer A schickt ein Paket zurück → bevor es im Lager landet, kauft Käufer B es auf FYNDDS → FYNDDS sagt DHL: "Neue Adresse" → Paket geht direkt zu Käufer B → Händler bekommt Geld ohne Lagerumweg.
